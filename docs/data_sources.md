# Data Sources

This document catalogues all input data, their locations, formats, and
provenance. It also describes how to reproduce the data pipeline from scratch.

---

| Data | Format | Dimensions | Location | Size |
|------|--------|-----------|----------|------|

| Clinical metadata | XLSX (multiple `CLIN_*` sheets) | ~~800 rows | S3 / local (see `config/settings.yaml`) | ~2 MB |
| GENCODE v45 GTF | GTF (gzipped) | ~63k gene records | GENCODE FTP | ~45 MB |

## 1. Beta Matrix

### Description

The beta matrix contains DNA methylation beta values for ~800 samples across
~4M CpG sites. Beta values range from 0 (unmethylated) to 1 (fully
methylated), with NaN indicating missing data.

### Format

- **File**: `beta_matrix.csv`
- **Delimiter**: comma
- **Header row**: First cell is empty (row-index column), followed by CpG
  column names in `chr{N}_{position}` format (e.g., `chr1_10469`)
- **First column**: Sample barcodes (row identifiers)
- **Coordinate system**: 1-based positions (GRCh38)
- **Missing values**: Empty string, `NA`, or `NaN`

### Location

Configured in `config/settings.yaml`:

```yaml
paths:
  beta_matrix: ${BETA_MATRIX_PATH:-data/raw/beta_matrix.csv}
```

### Provenance

The beta matrix is generated by the `the upstream beta-matrix generator` project, which
processes raw IDAT files from the Illumina MethylationEPIC array through a
pipeline including:

1. IDAT file reading
2. Preprocessing and normalization (functional normalization or noob)
3. Quality control filtering
4. Beta value computation

The `inspect-beta-generator` CLI command reads the generator source code to
verify conventions (CpG naming, coordinate system, delimiter, missing value
representation) without modifying any files.

```bash
inspect-beta-generator --generator-path 
```

### Derived outputs

| Output | Path | Description |
|--------|------|-------------|
| Per-chromosome Parquet | `data/processed/beta_chrom/beta_chr{N}.parquet` | 25 files (chr1-22, chrX, chrY, chrM), ~11 GB total |
| Dev subset | `data/dev/beta_subset.parquet` | 50 samples x 1000 CpGs |
| CpG list (dev) | `data/dev/cpg_list.txt` | Selected CpG IDs, one per line |
| Sample list (dev) | `data/dev/sample_list.txt` | Selected sample barcodes, one per line |

The per-chromosome Parquet format replaces the earlier single-file approach.
Each file contains all ~800 samples (rows) and all CpGs on that chromosome
(columns). Column names use the `chr{N}_{position}` format. The conversion
is performed by the `convert-beta` CLI command using a single-pass line-by-line
reader (see [architecture.md](architecture.md) for rationale).

## 2. Clinical Metadata

### Description

The clinical metadata workbook contains sample-level information across
multiple worksheets matching the `CLIN_*` pattern.

### Location

```yaml
paths:
  clinical_metadata: ${CLINICAL_METADATA_PATH:-data/raw/clinical_metadata.xlsx}
```

### Worksheet structure

Each `CLIN_*` worksheet contains (at minimum):

| Column | Aliases | Description |
|--------|---------|-------------|
| Barcode | `barcode`, `Barcode`, `sample_barcode`, `Sample_Barcode`, `Sample_ID`, `PD_Barcode` | Sample identifier matching beta matrix row names |
| Clinical category | `clinical_category`, `Clinical_Category`, `category`, `diagnosis`, `Group` | One of: `CRC`, `Control`, `polyps` |

The parser auto-detects column names from a list of known aliases (see
`load_clinical_metadata._BARCODE_ALIASES` and `_CATEGORY_ALIASES`).

### Processing

```bash
ingest-clinical -o data/processed/clinical_metadata.parquet
```

This command:

1. Discovers all `CLIN_*` worksheets in the workbook
2. Parses barcode and clinical category from each sheet
3. Normalizes barcode format and clinical category labels
4. Deduplicates by barcode (first occurrence wins)
5. Writes `data/processed/clinical_metadata.parquet`

### Output schema

| Column | Type | Description |
|--------|------|-------------|
| `barcode` | `Utf8` | Normalized sample barcode |
| `clinical_category` | `Utf8` | Normalized category: `CRC`, `Control`, or `polyps` |
| `source_sheet` | `Utf8` | Name of the worksheet this record came from |

### Gene-symbol-keyed pathway and GO mappings

In addition to the Ensembl-keyed Reactome mapping above, the pipeline produces
gene-symbol-keyed mapping files for use in enrichment analysis (Fisher exact
test and GSEA operate on gene symbols from the cohort comparison output):

| Output | Path | Description |
|--------|------|-------------|
| Reactome gene-symbol map | `data/external/reactome_gene_symbol_pathway.parquet` | Gene symbol to Reactome pathway ID/name mapping |
| GO gene-symbol map | `data/external/go_gene_symbol_annotations.parquet` | Gene symbol to GO term ID/name/namespace mapping |

These files are derived from the Ensembl-keyed sources by joining against
GENCODE gene annotations to resolve Ensembl IDs to HGNC symbols.

### CpG island context mapping

The `map_cpg_to_islands.py` module assigns each CpG site a genomic context
label relative to CpG islands:

| Context | Definition |
|---------|-----------|
| island | CpG falls within a CpG island boundary |
| shore | CpG is within 2 kb flanking an island |
| shelf | CpG is 2-4 kb from an island (flanking a shore) |
| open_sea | CpG does not fall in any of the above |

Output: `data/processed/cpg_island_context.parquet` with columns `cpg_id`,
`chromosome`, `position`, and `island_context`.

### Hypermethylation outputs

The `hypermethylation` CLI command produces:

| Output | Path | Description |
|--------|------|-------------|
| Per-gene thresholds | `data/processed/features/gene_thresholds.parquet` | 99th percentile of control betas per gene |
| Per-sample HMS | `data/processed/features/hypermethylation_scores.parquet` | Hypermethylation score (count of genes exceeding threshold) per sample |
| HMS summary | `data/processed/features/hms_summary.parquet` | Mean HMS by clinical category |

## 4. Reproducing the Data Pipeline from Scratch

### Prerequisites

1. Python 3.11+ with `epigraph` installed (`pip install -e ".[dev]"`)
2. Access to the beta matrix CSV (S3 or local)
3. Access to the clinical metadata XLSX (S3 or local)
4. TypeDB 3.x server running (for import steps)
5. Internet access for annotation downloads

### Step-by-step reproduction

The recommended approach is to use the pipeline orchestrator:

```bash
# Set paths if not using defaults
export BETA_MATRIX_PATH=/path/to/beta_matrix.csv
export CLINICAL_METADATA_PATH=/path/to/clinical_metadata.xlsx

# Run everything (dev mode for testing, production for full dataset)
run-pipeline --mode dev        # 50 samples, 1000 CpGs, <1 min
run-pipeline --mode production  # ~800 samples, 4M CpGs, ~30 min
```

Alternatively, run individual steps:

```bash
# 1. Inspect upstream generator (informational only)
inspect-beta-generator

# 2. Parse clinical metadata
ingest-clinical -o data/processed/clinical_metadata.parquet

# 3. Create dev subset for fast iteration
create-dev-subset --config config/dev_subset.yaml

# 4. Download and parse all annotations
build-annotations --config config/annotation_sources.yaml

# 5. (Manual) Download MSigDB files after registration
#    Place GMT files in data/external/msigdb/

# 6. Convert beta matrix to per-chromosome Parquet
convert-beta --csv data/raw/beta_matrix.csv --out-dir data/processed/beta_chrom/

# 7. Map CpGs to genes
map-cpg-genes \
  --genes data/external/genes.parquet \
  --cpg-source data/processed/beta_chrom/ \
  --output data/processed/cpg_gene_mapping.parquet

# 8. Import into TypeDB
import-typedb --data-dir data --step all

# 9. Aggregate features (chromosome-at-a-time for production)
derive-features \
  --beta-dir data/processed/beta_chrom/ \
  --cpg-gene-mapping data/processed/cpg_gene_mapping.parquet \
  --output-dir data/processed/features

# 10. Compute hypermethylation scores
hypermethylation \
  --gene-matrix data/processed/features/gene_features.parquet \
  --metadata data/processed/clinical_metadata.parquet \
  --output-dir data/processed/features

# 11. Run cohort comparisons
run-analysis \
  --feature-matrix data/processed/features/gene_features.parquet \
  --metadata data/processed/clinical_metadata.parquet \
  --output-dir data/processed/results

# 12. Pathway and GO enrichment
pathway-enrichment \
  --results data/processed/results/CRC_vs_Control.parquet \
  --output-dir data/processed/enrichment

# 13. Biomarker candidate ranking
biomarker-candidates \
  --results-dir data/processed/results/ \
  --output-dir data/processed/biomarkers
```

### Checksums and validation

To verify data integrity after reproduction:

1. Check that the clinical metadata has the expected number of samples per
   category (run `ingest-clinical` and review the summary output)
2. Check that the dev subset has exactly 50 samples and 1000 CpGs
3. Verify annotation record counts against the expected values in the table
   above
4. Run `import-typedb --step all` and verify the insertion counts match
   expectations
